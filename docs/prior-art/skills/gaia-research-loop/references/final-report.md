# Final Report Guidance

Use this reference when writing the final assessment report or the final
benchmark summary for a live eval.

## Chinese Scholarly Evidence Review

For `zh` runs, the final assessment report should read like a compact scholarly
review, not a command transcript.

Write for domain readers. Avoid Gaia/LKM/CLI/workflow language in the review
body except in a separate provenance or audit section. Use inline evidence refs
generated by the assessment artifact; do not invent citations.

Recommended structure:

```markdown
# <研究问题标题>

## 摘要

120-180 个中文字符左右，说明问题、证据方向、暂定结论和关键不确定性。

## 关键结论

- 4-6 条要点，每条都应对应 assessment 中的 evidence relation 或 limitation。

## 证据综述

按证据簇、方法路线、模型假设、观测指标或实验体系分节。每节说明：
已知什么、为什么重要、它支持/反对/限定了什么、仍缺什么。

## 竞争解释与限定条件

集中讨论反对、限定和削弱证据。不要把所有分歧写成“需要更多研究”。

## 后续关键检验

列出能改变判断的 targeted evidence needs，而不是泛泛的未来工作。

## 局限性

写科学局限性，例如共享校准锚点、模型依赖先验、误差预算不完整、样本
外推、定义不一致、观测覆盖不足等。
```

## Final Eval Summary

At the end of a benchmark/live eval, summarize:

- end-to-end elapsed time from the run envelope, plus recorded benchmark
  `summary.total_wall_seconds`; say plainly when the benchmark sum is only
  instrumented step time and excludes agent analysis, JSON writing, or recovery
  work;
- total token usage if available;
- number of search queries, raw results, paper leads, focuses, relations,
  candidate obligations, and candidate relations written;
- whether stop criteria recommends expand, assess another focus, promote, or
  stop;
- top 3 workflow bottlenecks or quality problems observed;
- path/state hygiene issues encountered and fixed, such as accidental nested
  package paths, parent-checkout writes, lost trace records, or open
  obligations;
- artifact paths, especially `$RUN`, `benchmark.json`, `trace.jsonl`,
  `evaluation_trace.md`, focus/assessment/stop reports, and analysis JSON.

## What Not To Do

- Do not paste a command transcript as the review.
- Do not expose credentials, signed URLs, or access keys.
- Do not claim token usage when the provider did not expose it.
- Do not present benchmark step-sum wall time as end-to-end eval duration.
- Do not treat search rank as confidence.
- Do not hide unresolved obligations; mark them resolved, deferred with
  rationale, or still blocking.
